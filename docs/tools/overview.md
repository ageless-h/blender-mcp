# Blender MCP 工具架构设计

## 设计哲学

**核心问题**：如何避免开发 1000+ 个工具？

**答案**：统一 CRUD 接口 + 类型参数化 + 通用操作执行器

## 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│              Blender MCP 完整工具集                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  核心工具（8个）                                                 │
│  ────────────────────────────────────────────────────────────   │
│  data.create      创建数据                                       │
│  data.read        读取数据（支持图像base64、评估后数据）         │
│  data.write       修改数据                                       │
│  data.delete      删除数据                                       │
│  data.list        列出数据                                       │
│  data.link        关联数据                                       │
│  operator.execute 执行操作（支持上下文覆盖）                     │
│  info.query       查询信息（状态/历史/反馈/截图）                │
│                                                                  │
│  可选工具（1个，默认禁用）⚠️                                     │
│  ────────────────────────────────────────────────────────────   │
│  script.execute   执行任意Python代码                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 压缩比

| 方案 | 工具数量 | 开发成本 | 维护成本 | AI理解难度 |
|-----|---------|---------|---------|-----------|
| 传统方式 | 1000+ | 极高 | 极高 | 困难 |
| 粗粒度 | 20 | 低 | 低 | 简单 |
| **统一CRUD** | **9** | **中** | **低** | **简单** |

**最终压缩比**：约 111:1

## 与 Blender API 的映射

```
┌─────────────────────────────────────────────────────────────────┐
│                    Blender Python API 架构                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  第1层：bpy.data（数据层）                                       │
│  └── 映射到：data.create / read / write / delete / list / link │
│                                                                  │
│  第2层：bpy.ops（操作层）                                        │
│  └── 映射到：operator.execute                                   │
│                                                                  │
│  第3层：bpy.context（上下文层）                                  │
│  └── 映射到：data.read/write(type="context") +                  │
│              operator.execute 的 context 参数                   │
│                                                                  │
│  元信息层：reports / history / stats                            │
│  └── 映射到：info.query                                         │
│                                                                  │
│  逃生舱：任意Python代码                                          │
│  └── 映射到：script.execute（可选，默认禁用）                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
