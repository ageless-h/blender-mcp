# Top-Level Layout

## Purpose
Define the repository root layout and responsibilities for each top-level directory.

## Layout
- `src/`: Core service code (single package root).
- `addon/`: Blender addon code and contract artifacts.
- `docs/`: Project documentation by domain.
- `openspec/`: OpenSpec changes, specs, and archives.
- `tests/`: Automated tests.
- `examples/`: Runnable examples and usage demos.
- `scripts/`: Developer scripts and maintenance utilities.

## Rules
- Each top-level directory MUST have a short README describing ownership and scope.
- New top-level directories require an update to this document.
